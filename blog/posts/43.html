So how did I do it? And what did I learn in the process?

So let's talk about how to do it. The only thing I paid for and bought, is a domain. No hosting, no SSL certificate. Github Pages will help with both of those.

1. Buy a domain

There are a gazillion number of websites that sell domain names. I went with one, randomly. hehe, mainly I saw a great deal for a year and bought it. So, you know next year this domain might change.

So I bought only the domain name (check the browser URL to see what domain I bought - might've changed when you read this ðŸ˜‚)

For the sake of an example, let's just say I bought whataweirddomainlel.com

2. Configure your DNS

On the website where you bought your domain, you'll get a basic interface to update DNS entries.

If you don't know, DNS, or a Domain Name Server is a cache that stores pointers from a domain name (eg. mrsauravsahu.github.io to its IP Address)

There are various types of DNS records, let's see what we need to configure to get this working...

2.1 Add an A record

An A record (an Address record) is a type of record that points a domain name to an IP Address. So add an A Record pointing whataweirddomainlel.com to the IP Address of Github Pages, where your site is hosted. You can find the Github Pages' IP Address here

This maps your Apex domain (your naked domain whataweirddomainlel.com to Github Pages' IP Address)

This helps users visit your site without the www subdomain.

2.2 Add a CNAME record

Since some people (should I say it, the word?) still use the www subdomain, when visiting websites, we'll have to redirect them to our Apex domain.

So we add a CNAME record, which maps one domain to another domain. And you must've guessed it, we need to map www.whataweirddomainlel.com to whataweirddomainlel.com

2.3 Wait and check if your DNS records Propagated

Now hastily keep checking websites like https://www.whatsmydns.net/, https://dnsmap.io/, and use the dig command to check if your DNS records propagated throughout the World. ðŸ˜‚

After waiting for an eternity, took me a day, you should see your DNS records propagated throughout the world.

2.4 Tell Github Pages - I need a custom domain!

Go to the settings on the repository where the site is on Github. In the section for Github Pages, write the custom domain.

Wait for Github to check if you've configured your DNS records correctly. Once Github verifies that you've added the A record properly, you'll be able to check the tick-box saying enforce HTTPS.

Now you have both HTTPS and a custom domain!
